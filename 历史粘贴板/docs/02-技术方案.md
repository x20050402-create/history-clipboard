# 02 · 技术方案

## 1. 技术栈

| 项目 | 选择 | 说明 |
| --- | --- | --- |
| 桌面框架 | Electron | 颜值好、生态成熟、资料多、易维护 |
| 界面 | 原生 HTML + CSS + JS | 卡片列表、搜索、淡蓝主题，无需额外框架 |
| 剪贴板 | Electron `clipboard` API + 定时轮询 | 捕获文字 / 图片 / 文件 |
| 本地存储 | better-sqlite3（SQLite） | 同步 API，简单稳定；存元数据 |
| 图片存储 | 文件形式存磁盘，库里只存文件名 | 避免数据库膨胀 |
| 系统托盘 | Electron `Tray` API | 托盘图标 + 右键菜单 |
| 打包 | electron-builder（NSIS） | 打成 Windows 安装包 .exe |

## 2. 整体架构（三层）

```
┌─────────────── 主进程 (src/main.js) ───────────────┐
│  窗口管理 / 托盘 / 设置读写 / 定时清理 / IPC 处理      │
│  ├── src/db.js              数据库读写、清理          │
│  └── src/clipboard-watcher.js  剪贴板轮询、去重、过滤  │
└──────────────────────┬─────────────────────────────┘
                       │ IPC（contextBridge 安全桥接）
┌──────────────────────┴─────────────────────────────┐
│  渲染进程 (src/renderer/)  界面与交互                  │
│  ├── index.html  结构                                │
│  ├── style.css   淡蓝主题                            │
│  └── renderer.js 列表渲染、搜索、卡片操作              │
└─────────────────────────────────────────────────────┘
        预加载 src/preload.js：仅暴露受控 API
```

## 3. 模块职责

- **main.js**：创建窗口/托盘；注册 IPC（取列表、复制、置顶、删除、读写设置）；启动监听；每小时清理；单实例锁。
- **db.js**：`init / insertClip / getClips / togglePin / deleteClip / cleanup` 等；图片删除时同步删文件。
- **clipboard-watcher.js**：定时轮询剪贴板，识别类型，去重（指纹），跳过敏感格式，写库后通知界面刷新。
- **preload.js**：用 `contextBridge` 暴露白名单 API 给界面（如 `getClips / copyClip / togglePin / deleteClip / getSettings / setRetention`）。
- **renderer**：从 preload API 取数据并渲染；监听 `clips-updated` 自动刷新。

## 4. 数据结构

数据库表 `clips`：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| id | INTEGER PK | 自增主键 |
| type | TEXT | `text` / `image` / `files` |
| content | TEXT | 文字内容；图片文件名；文件路径列表 JSON |
| preview | TEXT | 列表显示与搜索用的摘要 |
| created_at | INTEGER | 毫秒时间戳 |
| pinned | INTEGER | 0/1 是否置顶 |

排序规则：`ORDER BY pinned DESC, created_at DESC`。

## 5. 数据存放位置

- 目录：`%APPDATA%/剪贴历史/`（Electron `app.getPath('userData')`）
- 文件：`clip-history.db`（数据库）、`images/`（图片）、`settings.json`（设置）
- **全部在本机，不上传任何服务器。**

## 6. 安全约定

- `contextIsolation: true`、`nodeIntegration: false`
- 界面只能通过 preload 白名单 API 与主进程通信，不直接访问 Node / 文件系统
- 敏感剪贴板格式（如 `ExcludeClipboardContentFromMonitorProcessing`）直接跳过不记录

## 7. 已知技术点 / 风险

- Windows 复制文件的剪贴板格式（CF_HDROP）Electron 无直接 API，先用 `text/uri-list` 兜底，拿不到则不记录文件（阶段 2.4 重点验证）。
- 轮询间隔默认 800ms，兼顾灵敏与省电。
